# ClassroomFucker

班上的人天天拿一体机看一些特别糖的片，各种什么奶龙、锻刀大赛、华强卖瓜啥的，我被严重精神污染。
因此我写了一个这个项目，干扰一体机工作，这样再也没有人能够放那些唐人片了。

全项目使用AI完成，没人喜欢干查win32api这种黑奴工作。

## Prompt

```markdown
班上的人天天拿一体机看一些特别糖的片，各种什么奶龙、锻刀大赛、华强卖瓜啥的，我被严重精神污染。
我需要阻止学校内学生玩一体机的行为，这次不需要管理员权限。

依据以下课表帮我使用C++编写一个程序：

- 08:45 - 08:53
- 09:40 - 10:08
- 10:55 - 11:03
- 14:25 - 14:33
- 15:20 - 15:33
- 16:20 - 16:28
- 11:50 - 13:38 午休
- 17:15 - 18:28 晚休

程序全部功能如下：

1. 在课间时间段创建一个全屏、顶层、不断获取焦点的纯黑色窗口，要求不显示在任务栏上。
2. 在午休、晚休时间段，开机30秒后自动关机。
3. 添加开机自启项目，要求能够稳定地开机自启，后台静默运行。
4. 程序名伪装成ConclusionService.exe避免被发现。
5. 程序将从u盘内完成首次运行，运行之后立即拷贝到用户目录下并添加开机自启项目。
6. 程序首次运行，结束原有的（如果有）ConclusionService.exe，并且5分钟内，不执行操作1, 2。
7. 如果电脑有u盘插入，程序将会复制自身到u盘autorun.exe，如果没有则新建，有则清空写入。
8. 如果电脑有u盘插入，且u盘内存在iso等任何形式的系统映像文件，包括无盘符分区内文件，
   都随机更改几个字符以彻底损坏这些文件，阻碍学生重装系统。
9. 程序额外复制一份到D盘，方便随时运行。

编译使用g++，用64位编译。上次的exe提示缺失api-ms-win-crt-runtime-l1-1-0.dll，
是一个vsredist组件。有没有什么更稳定的静态链接方案。我追求极致64位兼容性。
-static-libstdc++ -static-libstdc++

还有什么疑问吗？
```
